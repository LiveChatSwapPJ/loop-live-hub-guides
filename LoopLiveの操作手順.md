# LoopLive 操作手順書

LoopLive Web アプリの操作手順を、画面・操作・結果確認の形で定義する。

---

## 1. 固定情報（画面名・権限・ダイアログ）

### 1-1. 画面のユーザー向け表示名と内部名・ルートの対応（一覧）

| ユーザー向け表示名 | 内部名（実装） | ルート |
|:---|:---|:---|
| ポータル（トップ） | `PortalPage` | `/` |
| サーバー管理 | `InstanceDashboard` | `/dashboard` |
| 配信（リアルタイム加工） | `StreamingPage` | `/streaming/:instanceId` |
| 写真フェイススワップ | `PhotoFaceSwapPage` | `/photo-face-swap` |
| 利用料金 | `UsageBillingPage` | `/usage-billing` |
| ユーザー管理 | `UserManagementPage` | `/users` |
| サーバーマスタ管理 | `ServerMasterPage` | `/master/servers` |
| プロフィール | `ProfilePage` | `/profile` |

※ルートを持たない UI（ダイアログ・ウィザード内ステップ・画面内エリア）は、下表のとおりユーザー向けの呼び方で統一する。

| ユーザー向け表示名 | 内部参照・備考 |
|:---|:---|
| サインアップ | Amplify Authenticator のサインアップ |
| 利用規約ダイアログ | `TermsAgreementDialog` |
| 確認ダイアログ | `ConfirmationDialog`（共通） |
| ユーザー詳細 | ダイアログ |
| ユーザー編集 | ダイアログ |
| サーバー登録 ステップ1 | 文言: `ステップ1: 登録するインスタンスの選択` |
| サーバー登録 ステップ2 | 文言: `ステップ2: 新規サーバー登録` |
| サーバー詳細 | ダイアログ |
| 利用履歴 | ダイアログ（利用料金から開く） |
| 処理履歴 | `PhotoFaceSwapPage` 内エリア |
| OBS連携 | `StreamingPage` 内・PC表示の `ObsController` 相当エリア |

### 1-2. ロールと行える操作

**内部コード（実装・手順表の「画面」列で使う場合）**

| コード | 行える操作の概要 |
|:---|:---|
| `GENERAL` | 配信開始/停止、写真フェイススワップ |
| `CREATOR` | 写真フェイススワップ |
| `OPERATOR` | ユーザー管理、利用料金確認 |
| `SYSTEM_ADMIN` | サーバーマスタ管理 |

**ユーザー向け表記（エンドユーザー向けガイドではこちらを使う）**

| 画面上の権限の呼び方 | 内部コード |
|:---|:---|
| 一般 | `GENERAL` |
| クリエイター | `CREATOR` |
| 運用 | `OPERATOR` |
| システム管理者 | `SYSTEM_ADMIN` |
| 利用不可 | （権限なし相当・表記は UI に合わせる） |

### 1-3. 確認ダイアログ（共通）

- キャンセル: `キャンセル`
- 確定: `はい`

### 1-4. 利用規約ダイアログ（サインアップ時）

- 同意前の文言: `利用規約を最後までスクロールしてご確認ください。`
- ボタン: `閉じる` / `同意する`

## 2. 手順書1（管理者向け）始め方（アカウント作成〜初期セットアップ）

### 2-1. 目標（完了状態）

- 新規ユーザーがログインできる
- 必要なロールが付与されている
- ユーザーが利用可能なサーバーを選べる

### 2-2. 手順（確定版）

| No | 画面 | 操作 | 結果確認 |
|---:|---|---|---|
| 1 | サインアップ | `LoopLive 利用規約` が表示されるので、`利用規約を最後までスクロールしてご確認ください。` が消えるまでスクロールする | `同意する` が有効になる |
| 2 | 利用規約ダイアログ | `同意する` を押す | サインアップ入力フォームが表示される |
| 3 | サインアップ | `グループID`（6文字英数字）を入力（表示されている値がある場合はその値を確認） | エラーが表示されない |
| 4 | サインアップ | `Email`（形式が正しい）と `パスワード`（8文字以上）を入力 | エラーが表示されない |
| 5 | サインアップ | `Cognito`標準の「アカウントを作る」ボタンを押す | ログイン後にポータル画面が表示される |
| 6 | `PortalPage` | `ユーザーを管理する` を押す（運用者/管理者権限がある場合） | ユーザー管理画面が表示される |
| 7 | `UserManagementPage` | 対象ユーザーのカードをクリック | `ユーザー詳細` ダイアログが表示される |
| 8 | `ユーザー詳細` | `編集` を押す | `ユーザー編集` ダイアログに切り替わる |
| 9 | `ユーザー編集` | `権限` を選択する（`利用不可`/`一般`/`クリエイター`/`運用`/`システム管理者`） | ドロップダウン選択が反映されている |
| 10 | `ユーザー編集` | `配信アカウント情報` で `配信プラットフォーム` / `ID` / `パスワード` を入力（行の追加は `+`） | 保存後に警告が出なくなる（次の手順で確認） |
| 11 | `ユーザー編集` | （`SYSTEM_ADMIN`でログインしている場合）`ソース画像` で `画像を設定する`（または `画像を変更する`）→ `画像を選択する` → `確定`（必要なら `クリア`） | 画像が表示される |
| 12 | `ユーザー編集` | `更新` を押す | ユーザー一覧に反映される |
| 13 | `PortalPage` | `配信を始める` または `写真でフェイススワップする` を押す | 対象機能の画面へ遷移する |
| 14 | `PortalPage` | `サーバーマスタ管理` を押す（`SYSTEM_ADMIN` の場合） | サーバーマスタ管理画面が表示される |
| 15 | `ServerMasterPage` | `サーバー登録` を押す | `ステップ1: 登録するインスタンスの選択` が表示される |
| 16 | `ステップ1` | 対象インスタンスのラジオを選択 → `次へ` | `ステップ2: 新規サーバー登録` が表示される |
| 17 | `ステップ2` | `グループID` を入力（必須）→ `単価(USD)` と `MEMO` を入力（必要に応じて）→ `登録` | サーバー一覧に追加される |
| 18 | `PortalPage` → `InstanceDashboard` | `配信を始める` → `サーバー管理` で対象サーバーが表示されることを確認 | 入室できる状態または起動状態が確認できる |

### 2-3. 入力制約（手順書に必ず記載）

- `グループID`: 6文字英数字
- `パスワード`: 8文字以上
- `ユーザー名`: 必須
- `Email`: 必須、形式チェックあり
- `サーバー登録`: `グループID` 必須

### 2-4. よくあるエラー（表示メッセージ）

- `グループIDは6文字である必要があります。`（サインアップ）
- `Eメールは必須です。`（サインアップ）
- `ユーザー名は必須です。`（ユーザー編集/登録）
- `Emailの形式が正しくありません。`（ユーザー編集/登録）
- `パスワードは8文字以上である必要があります。`（ユーザー編集/登録）
- `指定されたEメールアドレスは既に使用されています。`（サインアップ）
- `グループIDは必須です。`（サーバー登録）

## 3. 手順書2 リアルタイム顔交換加工（配信）

### 3-1. 目標（完了状態）

- `加工開始` から `加工停止` まで動作する
- `加工後の映像を見る` または OBS連携で加工映像を確認できる

### 3-2. 手順（確定版）

| No | 画面 | 操作 | 結果確認 |
|---:|---|---|---|
| 1 | `PortalPage` | `配信を始める` を押す | `サーバー管理` 画面が表示される |
| 2 | `InstanceDashboard` | 対象サーバーカードで `入室` を押す | `StreamingPage` が開く |
| 3 | `StreamingPage` → サーバー状態 | `サーバー状態` が `OFF` または `起動失敗` の場合、`起動する` を押す | 確認ダイアログが表示される |
| 4 | 確認ダイアログ | `はい` を押す | `サーバーを起動しています。このままお待ちください...` が表示される |
| 5 | `StreamingPage` | サーバー状態が `準備完了` になったのを確認する | `カメラ` 設定が有効になる（`カメラに接続` が押せる） |
| 6 | `StreamingPage` | `カメラに接続` を押す | プレビューにカメラ映像が表示される（またはデバイス選択が有効） |
| 7 | `StreamingPage` | `カメラ` の選択が必要な場合は、`カメラ` を選び直す | 画面の映像が切り替わる |
| 8 | `StreamingPage` | 鼻下加工の切替を必要に応じて行う（`鼻下を加工します` / `鼻下を加工しません`） | 文字表記が切り替わる |
| 9 | `StreamingPage` | `加工開始` を押す | 確認ダイアログが表示される |
| 10 | 確認ダイアログ | `はい` を押す | ボタンが `加工処理中…` になり、処理待ち状態になる |
| 11 | `StreamingPage` | ボタンが `加工停止` になるのを待つ | `加工停止` が表示される |
| 12 | `StreamingPage` |（必要に応じて）`加工後の映像を見る` を押す | 別タブで加工映像が表示される |
| 13 | `StreamingPage` | `加工停止` を押す | ボタンが `加工開始` に戻る |
| 14 | `StreamingPage` → サーバー状態 | サーバー状態が停止対象の場合、`停止する` を押す | 確認ダイアログが表示される |
| 15 | 確認ダイアログ | `はい` を押す | `サーバーを停止しています。このままお待ちください...` が表示され、状態が `OFF` になる |

### 3-3. OBS連携

| No | 画面 | 操作 | 結果確認 |
|---:|---|---|---|
| 1 | `StreamingPage`（PC表示） | `OBS WebSocket パスワード` を入力（空のままでも運用される場合がある） | 画面に入力が反映される |
| 2 | `StreamingPage` | `OBSにメディアソースを追加` を押す | UIに `OBSに接続完了` または `接続失敗` が表示される／OBS側で `LoopLiveからの映像` ソースが追加される |

### 3-4. 状態表示（手順書に必ず記載）

- `起動処理中` / `準備中` / `準備完了` / `使用中` / `停止処理中` / `OFF` / `起動失敗`

### 3-5. よくあるエラー（表示メッセージ）

- `インスタンス、ユーザー、またはリージョン情報がありません。`
- `クラウド側でサーバーが枯渇しており、一時的にサーバーを起動できません。別の地域のサーバーで試すか、時間を置いてから再度お試しください。`
- `サーバー起動リクエストの送信に失敗しました: ...`
- `加工処理がタイムアウトしました(300秒)。`
- `サーバー状態の確認に失敗しました。`
- `サーバーとの接続が切れました。`
- `カメラへのアクセスに失敗しました。`
- `サーバーへの接続に失敗しました: ...`
- OBS関連:
  - `OBS WebSocketパスワードが正しくありません。`
  - `OBS WebSocketパスワードは必須です。`
  - `OBSに接続できません。OBSが起動しているか、WebSocketサーバー設定を確認してください。`

## 4. 手順書3 写真フェイススワップ（写真）

### 4-1. 目標（完了状態）

- `フェイススワップする` が完了し、`処理履歴` に `完了` が表示される
- `結果` を `ダウンロード` できる

### 4-2. 手順（確定版）

| No | 画面 | 操作 | 結果確認 |
|---:|---|---|---|
| 1 | `PhotoFaceSwapPage` | `元の顔を選択` を押して、画像（JPG/PNG）をアップロード | プレビューが表示される |
| 2 | `PhotoFaceSwapPage` | `なりたい顔を選択` を押して、画像（JPG/PNG）をアップロード | プレビューが表示される |
| 3 | `PhotoFaceSwapPage` |（必要に応じて）`クリア` を押して選び直す | 画像がリセットされる |
| 4 | `PhotoFaceSwapPage` | `フェイススワップする` を押す | 確認ダイアログが表示される |
| 5 | 確認ダイアログ | `はい` を押す | `処理中です...` が表示される |
| 6 | `PhotoFaceSwapPage` |（自動反映または必要に応じて）`処理履歴` の最新行に追加されるのを待つ | ステータスが表示される |
| 7 | `処理履歴` | ステータスが `完了` になるまで待つ | `完了` のチップが表示される |
| 8 | `処理履歴` | `結果` のダウンロード（PC: 下矢印アイコン、スマホ: `ダウンロード`ボタン）を押す | 結果画像が表示/保存される |

### 4-3. ステータス表記（手順書に必ず記載）

- `待機中` / `処理中` / `完了` / `失敗`

### 4-4. 注意事項（手順書に固定記載）

- `※ フェイススワップを行った画像は、完了後24時間で自動的に削除されます。`

### 4-5. よくあるエラー（表示メッセージ）

- `画像が選択されていないか、ユーザー情報が取得できませんでした。`
- `エラーが発生しました: ...`
- `スワップ依頼が成功しました。完了までお待ちください。`（正常通知：次は`処理履歴`を見る）
- `スワップ履歴の取得に失敗しました。`
- `ダウンロードに失敗しました: ...`
