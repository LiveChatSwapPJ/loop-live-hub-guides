LoopLive 操作手順

LoopLive Web アプリの操作手順です。仕組みの説明は含みません。

手順書1（管理者向け）始め方（アカウント作成〜初期セットアップ）

目標: ログインできること、適切な権限が付与されていること、利用可能なサーバーを選べること。

以下は番号順に進めてください。各カードに「画面」「操作」「結果確認」と、そのステップの画面キャプチャをまとめています。PNG は docs/assets/guides/ に配置します。未配置のときは「画像未配置」と表示されます。ログインや個人情報が写り込まないようマスクしてください。

1サインアップ — 利用規約のスクロール

画面サインアップ

操作LoopLive 利用規約 が表示されるので、利用規約を最後までスクロールしてご確認ください。 が消えるまでスクロールする

結果確認同意する が有効になる

2利用規約ダイアログ — 同意

画面利用規約ダイアログ

操作同意する を押す

結果確認サインアップ入力フォームが表示される

3サインアップ — グループID

画面サインアップ

操作グループID（6文字英数字）を入力（表示されている値がある場合はその値を確認）

4サインアップ — Email とパスワード

画面サインアップ

操作Email（形式が正しい）と パスワード（8文字以上）を入力

5サインアップ — アカウント作成

画面サインアップ

操作「アカウントを作る」ボタンを押す

結果確認ログイン後にポータル画面が表示される

6ポータル — ユーザーを管理する

画面ポータル（トップ）

操作ユーザーを管理する を押す（運用者/管理者権限がある場合）

結果確認ユーザー管理画面が表示される

7ユーザー管理 — カード選択

画面ユーザー管理

操作対象ユーザーのカードをクリック

結果確認ユーザー詳細 ダイアログが表示される

8ユーザー詳細 — 編集

画面ユーザー詳細

操作編集 を押す

結果確認ユーザー編集 ダイアログに切り替わる

9ユーザー編集 — 権限

画面ユーザー編集

操作権限 を選択する（一般 / クリエイター / 運用 など）

結果確認ドロップダウン選択が反映されている

10ユーザー編集 — 加工アカウント情報

画面ユーザー編集

操作加工アカウント情報 で 加工プラットフォーム / ID / パスワード を入力（行の追加は +）

結果確認保存後に警告が出なくなる（次の手順で確認）

11ユーザー編集 — 更新

画面ユーザー編集

操作更新 を押す

結果確認ユーザー一覧に反映される

12ポータル — 加工または写真フェイススワップ

画面ポータル（トップ）

操作加工を始める（画面によっては 配信を始める など）または 写真でフェイススワップする を押す

結果確認対象機能の画面へ遷移する

13サーバー管理 — 対象サーバー確認

画面ポータル（トップ）→ サーバー管理

操作加工を始める → サーバー管理 で対象サーバーが表示されることを確認

結果確認入室できる状態または起動状態が確認できる

入力制約

  • グループID: 6文字英数字
  • パスワード: 8文字以上
  • ユーザー名: 必須
  • Email: 必須、形式チェックあり

手順書2 リアルタイム顔交換加工

目標: 加工開始 から 加工停止 まで動作すること。加工後の映像を見る または OBS 連携で加工映像を確認できること。

リアルタイム加工の基本操作（手順1〜15）のあと、OBS 連携（手順16〜17）を続けて記載しています。各カードで「画面」「操作」「結果確認」とキャプチャを照らし合わせてください。PNG は docs/assets/guides/ に配置します。本番または検証環境のログインとインスタンスが必要です。

1ポータル — 加工を始める

画面ポータル（トップ）

操作加工を始める（画面によっては 配信を始める など）を押す

結果確認サーバー管理 画面が表示される

2サーバー管理 — 入室

画面サーバー管理

操作対象サーバーカードで 入室 を押す

結果確認リアルタイム加工の画面が開く

3リアルタイム加工 — サーバー起動

画面リアルタイム加工・サーバー状態

操作サーバー状態 が OFF または 起動失敗 の場合、起動する を押す

結果確認確認ダイアログが表示される

結果確認サーバーを起動しています。このままお待ちください... が表示される

画面リアルタイム加工

操作サーバー状態が 準備完了 になったのを確認する

結果確認カメラ 設定が有効になる（カメラに接続 が押せる）

6リアルタイム加工 — カメラに接続

画面リアルタイム加工

結果確認プレビューにカメラ映像が表示される（またはデバイス選択が有効）

画面リアルタイム加工

画面確認ダイアログ

16OBS 連携 — WebSocket パスワード

補足（OBS 側）OBS Studio で ツール → WebSocket サーバー設定 を開き、WebSocket を有効にします。パスワードを設定するか、表示されているパスワードを確認し、LoopLive の入力欄と同じ値に揃えます。パスワードなしで使う場合は OBS も LoopLive も未設定に合わせます。

画面リアルタイム加工（PC 表示）

操作OBS WebSocket パスワード に、OBS で設定したパスワードを入力（空のままでも運用される場合がある）

結果確認画面に入力が反映される

17OBS 連携 — メディアソースを追加

画面リアルタイム加工

操作OBSにメディアソースを追加 を押す

結果確認UI に OBSに接続完了 または 接続失敗 が表示される／OBS 側で LoopLiveからの映像 ソースが追加される

状態表示

起動処理中 / 準備中 / 準備完了 / 使用中 / 停止処理中 / OFF / 起動失敗

手順書3 写真フェイススワップ（写真）

目標: フェイススワップする が完了し、処理履歴 に 完了 が表示されること。結果 を ダウンロード できること。

ログイン後の画面です。各カードに「画面」「操作」「結果確認」とキャプチャをまとめています。アップロードする画像は著作権・肖像権に注意し、手順書用はダミー画像の利用を推奨します。

1元の顔を選択

画面写真フェイススワップ

操作元の顔を選択 を押して、画像（JPG/PNG）をアップロード

結果確認プレビューが表示される

2なりたい顔を選択

画面写真フェイススワップ

操作なりたい顔を選択 を押して、画像（JPG/PNG）をアップロード

結果確認プレビューが表示される

3クリア

画面写真フェイススワップ

操作（必要に応じて）クリア を押して選び直す

結果確認画像がリセットされる

4フェイススワップする

画面写真フェイススワップ

操作フェイススワップする を押す

結果確認確認ダイアログが表示される

結果確認処理中です... が表示される

6処理履歴 — 待機

画面写真フェイススワップ

操作（自動反映または必要に応じて）処理履歴 の最新行に追加されるのを待つ

結果確認ステータスが表示される

7処理履歴 — 完了

画面処理履歴

操作ステータスが 完了 になるまで待つ

結果確認完了 のチップが表示される

8結果のダウンロード

画面処理履歴

操作結果 のダウンロード（PC: 下矢印アイコン、スマホ: ダウンロード ボタン）を押す

結果確認結果画像が表示/保存される

ステータス表記

注意事項

※ フェイススワップを行った画像は、完了後24時間で自動的に削除されます。

手順書4 利用料金を確認する方法（運用権限）

目標: 対象月の利用時間・請求金額の概要を把握し、サーバー別の起動・停止履歴とフォトスワップ利用枚数を確認できること。

運用権限が必要です。金額・ユーザー名はマスクしてください。手順1〜5をカードの説明とキャプチャで追ってください。

1ポータル — サーバー使用料を見る

画面ポータル（トップ）

操作サーバー使用料を見る を押す

結果確認利用料金 画面が表示される

2利用料金 — 合計利用時間・合計請求金額の見方

画面利用料金

説明同一画面の上部付近に、合計利用時間 と 合計請求金額 が並んで表示されます。対象月を変えるときは、見出し横の左右矢印（前の月 / 次の月）を使います。

結果確認月次の合計が表示される

3明細行をクリック

画面利用料金

操作明細テーブルでサーバー行をクリック

結果確認利用履歴 ダイアログが開く

4利用履歴ダイアログ

目的利用料金の明細からサーバーを開いたあとに表示されます。いつ・どのユーザーが・サーバーを起動したか／停止したかを時系列で確認するための一覧です。請求や運用の突合せ、利用状況の把握に使います。

画面利用履歴ダイアログ

操作列 ユーザー名 / 操作 / 日時 を読む。操作 には 起動 または 停止 が表示されます。

結果確認操作の流れを追える

5フォトスワップ使用

画面利用料金

操作フォトスワップ使用 の枚数を確認

結果確認XX枚 が表示される
