手順書1（管理者向け）始め方（アカウント作成〜初期セットアップ）

目標: ログインできること、適切な権限が付与されていること、利用可能なサーバーを選べること。

手順

No画面操作結果確認
1サインアップLoopLive 利用規約 が表示されるので、利用規約を最後までスクロールしてご確認ください。 が消えるまでスクロールする同意する が有効になる
2利用規約ダイアログ同意する を押すサインアップ入力フォームが表示される
3サインアップグループID（6文字英数字）を入力（表示されている値がある場合はその値を確認）エラーが表示されない
4サインアップEmail（形式が正しい）と パスワード（8文字以上）を入力エラーが表示されない
5サインアップCognito 標準の「アカウントを作る」ボタンを押すログイン後にポータル画面が表示される
6ポータル（トップ）ユーザーを管理する を押す（運用者/管理者権限がある場合）ユーザー管理画面が表示される
7ユーザー管理対象ユーザーのカードをクリックユーザー詳細 ダイアログが表示される
8ユーザー詳細編集 を押すユーザー編集 ダイアログに切り替わる
9ユーザー編集権限 を選択する（利用不可/一般/クリエイター/運用/システム管理者）ドロップダウン選択が反映されている
10ユーザー編集配信アカウント情報 で 配信プラットフォーム / ID / パスワード を入力（行の追加は +）保存後に警告が出なくなる（次の手順で確認）
11ユーザー編集（システム管理者でログインしている場合）ソース画像 で 画像を設定する（または 画像を変更する）→ 画像を選択する → 確定（必要なら クリア）画像が表示される
12ユーザー編集更新 を押すユーザー一覧に反映される
13ポータル（トップ）配信を始める または 写真でフェイススワップする を押す対象機能の画面へ遷移する
14ポータル（トップ）サーバーマスタ管理 を押す（システム管理者の場合）サーバーマスタ管理画面が表示される
15サーバーマスタ管理サーバー登録 を押すステップ1: 登録するインスタンスの選択 が表示される
16サーバー登録 ステップ1対象インスタンスのラジオを選択 → 次へステップ2: 新規サーバー登録 が表示される
17サーバー登録 ステップ2グループID を入力（必須）→ 単価(USD) と MEMO を入力（必要に応じて）→ 登録サーバー一覧に追加される
18ポータル（トップ） → サーバー管理配信を始める → サーバー管理 で対象サーバーが表示されることを確認入室できる状態または起動状態が確認できる

入力制約

  • グループID: 6文字英数字
  • パスワード: 8文字以上
  • ユーザー名: 必須
  • Email: 必須、形式チェックあり
  • サーバー登録: グループID 必須

よくあるエラー（表示メッセージ）

  • グループIDは6文字である必要があります。（サインアップ）
  • Eメールは必須です。（サインアップ）
  • ユーザー名は必須です。（ユーザー編集/登録）
  • Emailの形式が正しくありません。（ユーザー編集/登録）
  • パスワードは8文字以上である必要があります。（ユーザー編集/登録）
  • 指定されたEメールアドレスは既に使用されています。（サインアップ）
  • グループIDは必須です。（サーバー登録）

手順書2 リアルタイム顔交換加工（配信）

目標: 加工開始 から 加工停止 まで動作すること。加工後の映像を見る または OBS 連携で加工映像を確認できること。

手順

No画面操作結果確認
1ポータル（トップ）配信を始める を押すサーバー管理 画面が表示される
2サーバー管理対象サーバーカードで 入室 を押す配信（リアルタイム加工）の画面が開く
3配信（リアルタイム加工）・サーバー状態サーバー状態 が OFF または 起動失敗 の場合、起動する を押す確認ダイアログが表示される
4確認ダイアログはい を押すサーバーを起動しています。このままお待ちください... が表示される
5配信（リアルタイム加工）サーバー状態が 準備完了 になったのを確認するカメラ 設定が有効になる（カメラに接続 が押せる）
6配信（リアルタイム加工）カメラに接続 を押すプレビューにカメラ映像が表示される（またはデバイス選択が有効）
7配信（リアルタイム加工）カメラ の選択が必要な場合は、カメラ を選び直す画面の映像が切り替わる
8配信（リアルタイム加工）鼻下加工の切替を必要に応じて行う（鼻下を加工します / 鼻下を加工しません）文字表記が切り替わる
9配信（リアルタイム加工）加工開始 を押す確認ダイアログが表示される
10確認ダイアログはい を押すボタンが 加工処理中… になり、処理待ち状態になる
11配信（リアルタイム加工）ボタンが 加工停止 になるのを待つ加工停止 が表示される
12配信（リアルタイム加工）（必要に応じて）加工後の映像を見る を押す別タブで加工映像が表示される
13配信（リアルタイム加工）加工停止 を押すボタンが 加工開始 に戻る
14配信（リアルタイム加工）・サーバー状態サーバー状態が停止対象の場合、停止する を押す確認ダイアログが表示される
15確認ダイアログはい を押すサーバーを停止しています。このままお待ちください... が表示され、状態が OFF になる

よくあるエラー（表示メッセージ）

  • インスタンス、ユーザー、またはリージョン情報がありません。
  • クラウド側でサーバーが枯渇しており、一時的にサーバーを起動できません。別の地域のサーバーで試すか、時間を置いてから再度お試しください。
  • サーバー起動リクエストの送信に失敗しました: ...
  • 加工処理がタイムアウトしました(300秒)。
  • サーバー状態の確認に失敗しました。
  • サーバーとの接続が切れました。
  • カメラへのアクセスに失敗しました。
  • サーバーへの接続に失敗しました: ...
  • OBS 関連:
    • OBS WebSocketパスワードが正しくありません。
    • OBS WebSocketパスワードは必須です。
    • OBSに接続できません。OBSが起動しているか、WebSocketサーバー設定を確認してください。

手順書3 写真フェイススワップ（写真）

目標: フェイススワップする が完了し、処理履歴 に 完了 が表示されること。結果 を ダウンロード できること。

手順

No画面操作結果確認
1写真フェイススワップ元の顔を選択 を押して、画像（JPG/PNG）をアップロードプレビューが表示される
2写真フェイススワップなりたい顔を選択 を押して、画像（JPG/PNG）をアップロードプレビューが表示される
3写真フェイススワップ（必要に応じて）クリア を押して選び直す画像がリセットされる
4写真フェイススワップフェイススワップする を押す確認ダイアログが表示される
5確認ダイアログはい を押す処理中です... が表示される
6写真フェイススワップ（自動反映または必要に応じて）処理履歴 の最新行に追加されるのを待つステータスが表示される
7処理履歴ステータスが 完了 になるまで待つ完了 のチップが表示される
8処理履歴結果 のダウンロード（PC: 下矢印アイコン、スマホ: ダウンロードボタン）を押す結果画像が表示/保存される

画面キャプチャ

写真フェイススワップはログイン後の画面です。アップロードする画像は著作権・肖像権に注意し、手順書用はダミー画像の利用を推奨します。

1元の顔を選択
2なりたい顔を選択
3クリア
4フェイススワップする
5確認ダイアログ
6処理履歴 — 待機
7処理履歴 — 完了
8結果のダウンロード

ステータス表記

待機中 / 処理中 / 完了 / 失敗

注意事項

※ フェイススワップを行った画像は、完了後24時間で自動的に削除されます。

よくあるエラー（表示メッセージ）

  • 画像が選択されていないか、ユーザー情報が取得できませんでした。
  • エラーが発生しました: ...
  • スワップ依頼が成功しました。完了までお待ちください。（正常通知：次は処理履歴を見る）
  • スワップ履歴の取得に失敗しました。
  • ダウンロードに失敗しました: ...
